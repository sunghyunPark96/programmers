지역 변수 (Local Variables):
함수 내부에서 선언되며, 해당 함수 블록 내에서만 접근 가능한 변수입니다.
함수가 호출될 때 생성되고, 함수가 종료될 때 소멸됩니다.
다른 함수 내에서 동일한 이름의 지역 변수를 사용할 수 있습니다.
해당 함수의 스코프에서만 유효합니다.
스택 메모리에 저장됩니다.
함수의 매개변수도 지역 변수의 일종으로 간주됩니다.
  
전역 변수 (Global Variables):
함수 외부에서 선언되며, 프로그램 어디에서나 접근 가능한 변수입니다.
프로그램이 시작될 때 생성되고, 프로그램이 종료될 때 소멸됩니다.
여러 소스 파일에서 공유할 수 있습니다.
전역 스코프에 속하므로 프로그램 어디에서든 참조 가능합니다.
데이터 섹션 또는 BSS 섹션에 저장될 수 있습니다.
  
정적 지역 변수 (Static Local Variables):
함수 내부에서 static 키워드와 함께 선언되며, 해당 함수 블록 내에서만 접근 가능합니다.
함수가 호출될 때 생성되고, 프로그램이 종료될 때 소멸됩니다.
함수가 호출될 때마다 초기화되지 않고, 이전 값이 유지됩니다.
함수의 스코프에서만 유효하지만, 유지되는 값이 특징입니다.
  
정적 전역 변수 (Static Global Variables):
함수 외부에서 static 키워드와 함께 선언되며, 프로그램 어디에서나 접근 가능합니다.
프로그램이 시작될 때 생성되고, 프로그램이 종료될 때 소멸됩니다.
다른 소스 파일에서 해당 변수를 사용할 수 없으며, 파일 내에서만 유효합니다.
전역 스코프에 속하지만, 유지되는 값이 특징입니다.
  
레지스터 변수 (Register Variables):
register 키워드와 함께 선언되며, 변수를 레지스터에 저장하도록 컴파일러에게 제안합니다.
레지스터는 컴파일러에 의해 선택되므로 반드시 레지스터에 저장되는 것은 아닙니다.
지역 변수로만 사용할 수 있으며, 주소를 가질 수 없습니다.
레지스터에 저장되므로 메모리 접근 시간이 빨라지지만, 제약사항이 많습니다.
